# Dashboard 双层布局优化 — 需求

## 功能需求

### FR1: 多项目概览
- 系统应在上层显示所有项目卡片
- 卡片应水平滚动排列
- 每个卡片显示项目名称、阶段、进度、活动时间

### FR2: 项目切换
- 用户点击卡片后，下层详情应切换到对应项目
- 切换时应播放淡入动画

### FR3: 布局拖动
- 用户可拖动概览/详情分割线调整比例（15%-75%）
- 用户可拖动三栏分割线调整列宽（每列最小 10%）
- 拖动时应显示实时比例提示

### FR4: 布局记忆
- 系统应持久化用户的布局设置
- 刷新页面后应恢复上次的布局

### FR5: 重置布局
- 用户可点击按钮恢复默认布局

## 用户场景

### 场景 1: 快速浏览项目状态
**Given** 用户打开 Dashboard
**When** 查看上层概览区域
**Then** 能同时看到所有项目的关键状态信息

### 场景 2: 深入查看单个项目
**Given** 用户在概览区域
**When** 点击某个项目卡片
**Then** 下层详情显示该项目的完整信息

### 场景 3: 自定义布局比例
**Given** 用户对默认布局不满意
**When** 拖动分割线调整
**Then** 布局按预期调整，刷新后保持

### 场景 4: 恢复默认布局
**Given** 用户调整布局后想恢复默认
**When** 点击重置按钮
**Then** 布局恢复为概览 30% + 三栏 1:1:1

## 非功能需求

### NFR1: 性能
- 拖动响应延迟 < 50ms
- 卡片切换动画流畅（≥ 30fps）

### NFR2: 兼容性
- 支持窗口缩放自适应
- 最小窗口宽度 1024px

### NFR3: 可用性
- 拖动热区至少 4px
- 拖动时有明显视觉反馈（颜色变化）
- 比例提示清晰可读

### NFR4: 可靠性
- 布局设置持久化成功率 100%
- 异常情况自动恢复默认值
