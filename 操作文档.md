> **基于**: Superpowers + OpenSpec + GSD 真实源码
> **新增**: 原型图分析、大模块拆分、框架隐形规则扫描、实体继承规范

## 一、这是什么？

SillySpec 是可以直接在 Claude Code 里用的 slash commands，融合了三个顶级 AI 编码工具的核心能力：

| 来源 | 借鉴能力 | GitHub ⭐ |
|---|---|---|
| **Superpowers** | TDD 纪律、子代理并行开发、精准规划 | 99.6K |
| **OpenSpec** | 变更管理、规范沉淀、自由思考模式 | 32.5K |
| **GSD** | 棕地扫描、绿地初始化、状态恢复 | 36.4K |

**不是参考文档，是真命令。** 在 Claude Code 里输入 `/sillyspec:` 就能看到完整列表。

## 三、安装

### macOS / Linux

```bash
curl -fsSL https://raw.githubusercontent.com/q512426816/sillyspec/main/scripts/init.sh | bash
```

### Windows PowerShell

```powershell
powershell -c "irm https://raw.githubusercontent.com/q512426816/sillyspec/main/install.ps1 | iex"
```

安装后重新打开终端，启动 Claude Code：

```bash
claude --dangerously-skip-permissions
```

> 💡 **推荐使用跳过权限模式。** SillySpec 的命令会频繁执行 `git commit`、文件读写、校验脚本等操作，停下来批准会失去意义。

### 安装参数

| 参数 | 说明 |
|---|---|
| `--tool <name>` | 指定目标工具（claude/claude_skills/cursor/codex/opencode/openclaw） |
| `--workspace` / `-w` | 启用多项目工作区模式 |
| `--dir <path>` / `-d <path>` | 指定安装目录（默认当前目录） |

不传 `--tool` 时自动检测已安装的工具。

### 支持的 AI 工具

| 工具 | 输出格式 |
|---|---|
| Claude Code (commands) | `.claude/commands/sillyspec/` |
| Claude Code (skills) | `.claude/skills/sillyspec-<name>/` |
| Cursor | `.cursor/commands/` |
| Codex | `~/.agents/skills/` |
| OpenCode | `.opencode/skills/` |
| OpenClaw | `.openclaw/skills/` |

## 四、入口选择

### 绿地项目（空目录）

```bash
/sillyspec:init
```

深度提问：项目目的、核心功能、技术偏好、约束。生成 3 份文档。

### 棕地项目（已有代码）

```bash
/sillyspec:scan
```

交互式引导扫描，AI 会逐步问你：
1. 检查已有文档 → "上次扫描 3 天前，刷新吗？"
2. 选择扫描模式 → 快速 ⚡ / 深度 🔍
3. 选择扫描范围 → 留空全扫，或指定模块
4. 确认排除目录 → 默认排除 node_modules 等，可追加
5. 确认扫描计划 → 开始执行

**快速扫描**生成 3 份文档，**深度扫描**生成 7 份文档。

### 大模块（原型图/多页面）

```bash
/sillyspec:brainstorm
```

直接贴截图/原型图，AI 分析后：
1. 提取页面结构、表单字段、交互流程
2. 如果页面过多（>5），建议拆分为多个阶段
3. 生成 MASTER.md 记录整体方向
4. 逐阶段执行：brainstorm → propose → plan → execute → verify
5. 全部完成后统一归档

### 自由思考（随时可用）

```bash
/sillyspec:explore "要不要用 PostgreSQL 还是 SQLite"
```

## 五、完整工作流教程

### 场景 A：棕地项目添加功能

```
/sillyspec:scan         → 生成 7 份代码库文档
/sillyspec:brainstorm   → 逐个提问 → 设计方案
/sillyspec:propose      → 生成 4 份规范文件
/sillyspec:plan         → 精确实现计划
/sillyspec:execute      → TDD 执行
/sillyspec:verify       → 对照规范检查
/sillyspec:archive      → 规范沉淀
```

### 场景 B：大模块开发（原型图驱动）

```
1. /sillyspec:brainstorm "做奖惩台账功能"
   → 贴 20 张原型截图
   → AI 分析后建议拆分为 4 个阶段
   → 生成 MASTER.md

2. /sillyspec:brainstorm reward-punishment/stage-1
   → 阶段 1 的详细设计（自动读 MASTER.md）

3. /sillyspec:propose reward-punishment/stage-1
   → 生成阶段 1 的规范（保存到 stages/stage-1/）

4. /sillyspec:plan → /sillyspec:execute → /sillyspec:verify

5. /sillyspec:brainstorm reward-punishment/stage-2
   → 阶段 2 基于 MASTER.md + 阶段 1 的经验继续

6. 重复直到全部完成 → /sillyspec:archive reward-punishment
```

**中断恢复：**
```bash
/sillyspec:handoff    # 保存状态
/sillyspec:resume     # 恢复，自动显示各阶段进度
```

### 场景 C：工作区模式（前后端分离）

```bash
curl -fsSL ... | bash -s -- --workspace
/sillyspec:workspace add     # 添加子项目
/sillyspec:scan              # 遍历所有子项目扫描
/sillyspec:brainstorm "需求"  # 自动问属于哪个子项目
```

## 六、命令参考

### 核心流程

| 命令 | 参数 | 说明 |
|---|---|---|
| `/sillyspec:init` | 无 | 绿地项目：深度提问→需求→路线图 |
| `/sillyspec:scan` | `[区域] [--deep]` | 棕地扫描：交互式引导，快扫 3 份/深扫 7 份文档 |
| `/sillyspec:brainstorm` | `"想法"` 或 `<change>/stage-N` | 需求探索：支持原型图分析，大模块拆分 |
| `/sillyspec:propose` | `<name>` 或 `<change>/stage-N` | 生成 proposal+design+tasks |
| `/sillyspec:plan` | 无 | 实现计划：精确文件路径+代码 |
| `/sillyspec:execute` | `[范围]` | TDD 执行：子代理+两阶段审查 |
| `/sillyspec:verify` | 无 | 验证：对照规范+测试套件 |
| `/sillyspec:archive` | `[name]` | 归档：规范沉淀 |

### 辅助工具

| 命令 | 参数 | 说明 |
|---|---|---|
| `/sillyspec:status` | 无 | 查看进度 |
| `/sillyspec:continue` | 无 | 自动下一步 |
| `/sillyspec:explore` | `"话题"` | 自由思考：画图、讨论、调研 |
| `/sillyspec:quick` | `"任务"` | 快速模式：跳过完整流程 |
| `/sillyspec:handoff` | 无 | 保存状态（支持大模块阶段信息） |
| `/sillyspec:resume` | 无 | 恢复工作（支持大模块阶段进度表） |
| `/sillyspec:workspace` | `add/remove/status/info` | 工作区管理 |
| `/sillyspec:export` | `<name>` | 导出成功方案为模板 |

## 七、可靠性保障

### Prompt 层（软约束）

- **锚定确认** — propose/plan/execute/verify 执行前强制确认读过规范文件
- **Hard Gate 自检** — 关键命令生成文件后强制检查格式

### Shell 层（硬约束）

- **validate-proposal.sh** — 校验 propose 输出的 4 份文件
- **validate-plan.sh** — 校验 plan 输出的格式和内容
- **validate-scan.sh** — 校验 scan 输出的文档完整性
- **validate-all.sh** — 综合校验

### 扫描层（防幻觉）

scan 阶段执行三道强制扫描：

| 扫描 | 防什么 | 检测内容 |
|---|---|---|
| 数据库 Schema | 编造表名/字段名 | Prisma/SQLAlchemy/Django/TypeORM/MyBatis 等 |
| 框架隐形规则 | SQL 违反框架行为 | 多租户拦截器、逻辑删除、审计字段、查询增强 |
| 实体继承规范 | 新表漏掉通用字段 | BaseModel/BaseEntity 的通用列定义 |

覆盖框架：Java（MyBatis/JPA）、Django、SQLAlchemy、Prisma、Go GORM、TypeORM、Rails、Laravel。

## 八、文件生命周期

### 绿地流程

```
init → brainstorm → propose → plan → execute → verify → archive
 ↓        ↓          ↓        ↓       ↓       ↓       ↓
3 文件   设计方案   4 文件   计划    代码    检查    归档
```

### 棕地流程

```
scan → brainstorm → propose → plan → execute → verify → archive
 ↓        ↓          ↓        ↓       ↓       ↓       ↓
7+1文档  设计方案   4 文件   计划    代码    检查    归档
```

### 大模块流程

```
brainstorm(多图) → MASTER.md
  → stage-1: brainstorm → propose → plan → execute → verify
  → stage-2: brainstorm → propose → plan → execute → verify
  → ... → archive
```

目录结构：
```
.sillyspec/changes/<name>/
├── MASTER.md                # 主变更（整体方向、拆分计划）
├── stages/
│   ├── 01-list/
│   │   ├── proposal.md
│   │   ├── design.md
│   │   ├── tasks.md
│   │   └── plan.md
│   └── 02-form/
│       └── ...
```

### 关键文件关系

```
design.md（做什么）  → propose 产出
tasks.md（哪些任务）  → propose 产出，执行时勾进度
    ↓
plan.md（怎么做）   → plan 产出，执行主文档
    ↓
execute              → 按 plan 执行，用 tasks.md 追踪进度
```

## 九、常见问题

### Q: 安装后看不到 `/sillyspec:*` 命令？
重新打开终端。Claude Code 只在启动时加载 `.claude/commands/`。

### Q: scan 扫描大项目很慢？
交互式引导会问你要扫哪些区域。也可以传参跳过交互：`/sillyspec:scan api --deep`

### Q: 大模块的原型图太多怎么办？
brainstorm 会自动建议拆分，每个阶段 3-5 张图，逐个执行。

### Q: 电脑关机了，下次能恢复吗？
`/sillyspec:handoff` 保存状态后关机，下次 `/sillyspec:resume` 恢复。大模块模式会显示各阶段进度表。

### Q: 和分别装三个工具有什么区别？
| 维度 | 分别装 | SillySpec |
|---|---|---|
| 安装 | 3 次 | 1 个脚本 |
| 命令 | 3 套 | 1 套 `/sillyspec:*` |
| 流程串联 | 手动 | 每个命令结尾提示下一步 |
| 绿地/棕地 | 各自处理 | `init` / `scan` 统一入口 |
| 多项目 | 不支持 | `workspace` 工作区模式 |
| 大模块 | 不支持 | MASTER.md + 分阶段执行 |
| 原型图 | 不支持 | brainstorm 原型分析 |
| 防幻觉 | 各自不同 | 三道扫描防线 |

### Q: 可以只用一部分命令吗？
可以。不需要某个命令，删掉对应的 `.md` 文件即可。
